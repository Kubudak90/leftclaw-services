## Phase 1: x402 API Server

### Architecture
```
Agent → POST /api/consult (x402 payment: $20 USDC)
     → 402 Payment Required (first request)
     → Agent pays via x402 facilitator
     → Agent retries with payment proof
     → Server validates payment, creates job on-chain, runs consultation
     → Returns consultation result (build plan)
```

### Endpoints
| Endpoint | Price | Description |
|---|---|---|
| `POST /api/consult/quick` | $20 | Quick consultation — send project description, get build plan |
| `POST /api/consult/deep` | $30 | Deep consultation — detailed architecture review |
| `POST /api/audit` | $200 | Smart contract audit — send contract address or source |
| `POST /api/qa` | $50 | QA report — send dApp URL, get quality report |
| `GET /api/services` | Free | List available services and prices |
| `GET /api/job/:id` | Free | Check job status and results |

### Request/Response Format
```json
// POST /api/consult/quick
{
  "description": "I want to build a token dashboard for CLAWD",
  "context": "optional additional context, links, etc"
}

// Response (after payment)
{
  "jobId": 42,
  "status": "processing",
  "estimatedTime": "5-10 minutes",
  "resultUrl": "/api/job/42"
}

// GET /api/job/42 (when complete)
{
  "jobId": 42,
  "status": "completed",
  "result": {
    "buildPlan": "...",
    "gistUrl": "https://gist.github.com/...",
    "recommendedService": "BUILD_DAILY"
  }
}
```

### Tech Stack
- **Next.js API routes** (already in the project)
- **@x402/next** middleware for payment gating
- **x402 facilitator** — use Coinbase's hosted facilitator or run our own
- **Worker assignment** — auto-assign to available worker bot, or round-robin
- **On-chain job creation** — after payment confirmed, post job to contract via deployer-3

### x402 Payment Flow
1. Agent sends request without payment → gets `402 Payment Required` with payment details
2. Payment details include: amount ($20 USDC), network (Base), facilitator URL
3. Agent signs USDC payment via x402 client SDK
4. Agent retries with `X-Payment` header containing signed payment
5. Server validates via facilitator → payment settles on Base
6. Server creates on-chain job, assigns worker, returns job ID
7. Worker bot processes the job (consultation/audit/QA)
8. Result available at `/api/job/:id`

### Implementation Steps
1. `npm install @x402/core @x402/next @x402/evm`
2. Create API routes with x402 middleware
3. Wire up job creation → worker assignment → result delivery
4. The payment goes directly to our wallet (USDC on Base)
5. We can optionally also post the job on-chain for transparency

## Open Questions

1. **Facilitator:** Use Coinbase's hosted facilitator or self-host? (Coinbase's is simpler to start)
2. **Auto-accept:** Should x402 jobs auto-assign to a worker, or go through the normal OPEN → accept flow?
3. **Payment destination:** USDC goes to our wallet directly (x402) vs. through the smart contract? Simplest: direct to wallet, log job on-chain separately.
4. **Async vs sync:** Consultations could be:
   - **Sync:** Agent waits, bot processes, returns result in same HTTP response (with long timeout)
   - **Async:** Return job ID immediately, agent polls for result
   - Recommendation: **Async** with webhook support
5. **A2A protocol:** Should we also add an A2A (Agent-to-Agent) endpoint? The 8004 spec supports it.
